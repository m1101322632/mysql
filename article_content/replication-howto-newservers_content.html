17.1.1.7 在新的主服务器和从服务器上建立主从同步

建立主从同步最简单且最直接的方法时使用新的主服务器和从服务器。

你同样可以使用这个方法来处理新服务器建立主从同步且有已存在的备份数据要导入到数据库中的情况。通过将数据导入进新的主服务器，数据会自动的同步到 从服务器上。

在新的主服务器和从服务器上建立主从同步：

  1. 在MySQL主服务器上做必须的属性配置。详见章节17.1.1.1，“设置主从同步中的主服务器的配置”。

  2. 开启主服务器。

  3. 建立一个单独的用户。详见章节 17.1.1.3，“为主从同步创建独立的用户”。

  4. 获取主服务器状态信息。详见章节 17.1.1.4，“获取主从同步主服务器二进制日志坐标”。

  5. 在主服务器上，释放读锁： 

    Press CTRL+C to copy
     
    mysql> UNLOCK TABLES;

  6. 在从服务器上，编辑MySQL配置信息。详见章节 17.1.1.2，“配置主从同步从服务器信息”。

  7. 启动从服务器。

  8. 在从服务器上通过执行一个CHANGE MASTER TO 命令来设置主服务器配置信息。详见章节 17.1.1.10，“在从服务器上设置主服务器的配置信息”。

在每一个从服务器上执行设置步骤。

因为 Because there is no data to load or exchange on a new server configuration you do not need to copy or import any information.

If you are setting up a new replication environment using the data from a different existing database server, you will now need to run the dump file generated from that server on the new master. The database updates will automatically be propagated to the slaves: 

Press CTRL+C to copy
 
shell> mysql -h master < fulldb.dump
